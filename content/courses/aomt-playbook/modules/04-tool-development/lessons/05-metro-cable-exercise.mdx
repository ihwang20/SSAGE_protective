# Metro Cable Exercise: Tool Development

Translate Metro Cable's highest-priority opportunity into a solution design, select the right AI approach, and generate the MVP backlog.

<Callout type="concept" title="Where You Are in the Engagement">
The prioritized roadmap is approved. The executive committee has green-lighted the AI Skill-Match Dispatch opportunity as Horizon 1. Your job is to design the solution, select the right AI approach, and structure the MVP backlog so the development team can start sprinting immediately.
</Callout>

## Part 1: Choose the Solution Type

<DecisionPoint
  scenario="You are designing Metro Cable's AI Skill-Match Dispatch solution. Coordinators currently make all assignment decisions manually. The pilot will run in three zones with volunteer coordinators who are open to new tools. What is the right solution type for the initial deployment?"
  options={[
    {
      id: "a",
      label: "Agentic — the AI assigns technicians to jobs automatically without coordinator approval.",
      outcome: "Too aggressive for an initial deployment. Coordinator trust has not been established, the model has not been validated in production, and errors (wrong technician assigned) could cascade into SLA breaches. Agentic mode should be earned through demonstrated accuracy, not assumed at launch."
    },
    {
      id: "b",
      label: "Assistive — the AI recommends assignments and coordinators approve, modify, or reject.",
      outcome: "Correct for the pilot. Assistive mode builds trust by making the AI's reasoning visible, allows coordinators to catch model errors during the calibration period, and generates real-world feedback data for model improvement. A clear accuracy threshold (e.g., ≥85% acceptance rate) can define when to evaluate upgrading to agentic.",
      recommended: true
    },
    {
      id: "c",
      label: "Automation — rule-based assignment logic that handles simple jobs without AI.",
      outcome: "This foregoes the tier-matching intelligence that makes the solution valuable. Rule-based automation cannot handle the complexity of matching certification tiers to job types across dynamic zone conditions — that's exactly the problem that requires AI."
    }
  ]}

## Part 2: Write the MVP User Stories

<AIExercise
  scenario="Metro Cable's dispatch coordinators currently assign jobs by scanning a spreadsheet of available technicians, checking zone and certification tier, and making assignments based on experience. The new AI Skill-Match Dispatch system must integrate with the existing dispatch platform and certification database. The MVP will run in Assistive mode — the AI recommends, the coordinator approves."
  instructions="Write an Epic and four MVP User Stories for the AI Skill-Match Dispatch prototype. Follow the format: 'As a [role], I want [capability] so that [benefit].' Include acceptance criteria for each story. Focus on the core recommendation and approval workflow — scope the MVP tightly."
